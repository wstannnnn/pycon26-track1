# Data Provenance

This project grounds pathway and skills recommendations in the local SkillsFuture job and skills workbooks stored in `data/`.

## Source Files

| File | Purpose in the product |
| --- | --- |
| `data/jobsandskills-skillsfuture-skills-framework-dataset.xlsx` | Source workbook for job roles, sectors, tracks, critical work functions, key tasks, role-to-skill mappings, skill descriptions, proficiency levels, and knowledge/ability items. |
| `data/jobsandskills-skillsfuture-unique-skills-list.xlsx` | Source workbook for canonical unique skill names and descriptions. |
| `data/jobsandskills-skillsfuture-tsc-to-unique-skills-mapping.xlsx` | Source workbook that maps technical skills and competencies to the unique skills taxonomy. |
| `data/processed/skills_framework_joined_roles.jsonl` | Generated role-level records used for retrieval. Each line combines role description, performance expectation, tasks, skills, proficiency metadata, and searchable text. |
| `data/processed/jobsandskills-skillsfuture-unique-skills-list.json` | Generated JSON version of the unique skills workbook for reliable indexing. |

## Transformation Flow

The script `apps/backend/scripts/join_skills_framework_workbook.py` joins denormalized SkillsFuture workbook tabs into one structured record per job role.

It reads:

- `Job Role_Description` for sector, track, job role, role description, and performance expectation.
- `Job Role_CWF_KT` for critical work functions and key tasks.
- `Job Role_TCS_CCS` for role-to-skill and proficiency mappings.
- `TSC_CCS_Key` and `TSC_CCS_Key_Retired` for active and retired skill definitions.
- `TSC_CCS_K&A` for proficiency descriptions plus knowledge and ability items.

Each output record includes a `concatenated_description` field so ChromaDB can retrieve role, task, and skill evidence from one joined view.

## Retrieval Flow

The backend vector client in `apps/backend/app/clients/vector_db.py` indexes:

- Role records as `record_type=job_role`.
- Role-skill mappings as `record_type=role_skill`.
- Key tasks as `record_type=key_task`.
- Unique skills in a separate SkillsFuture unique skills collection.

The learner analysis endpoint searches role evidence first, then supporting role-skill/task records, then unique skills. The frontend displays the returned similarity evidence, match score, record type, and source file so users can see why a recommendation was made.

## Current Limitations

- The vector embedding is a local deterministic text-hash embedding, suitable for offline demo retrieval but not a production semantic model.
- Recommendations are generated by a configured local LLM from retrieved evidence; they should be treated as guidance, not official career advice.
- Source files are local snapshots. Refreshing the workbooks requires rerunning the join script and vector indexing endpoints.

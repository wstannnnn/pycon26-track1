# C4 Architecture Diagrams

These diagrams describe the Job and Skills Track application at C4 levels C1 and C2.

## C1: System Context

```mermaid
flowchart LR
    learner["Learner / Career Switcher<br/><br/>Explores career pathways, uploads resume text, and reviews skill recommendations."]
    admin["Product Team / Demo Operator<br/><br/>Indexes local SkillsFuture data and checks retrieval health."]

    system["Job and Skills Track<br/><br/>Career pathway and skills intelligence product that combines profile input, SkillsFuture evidence, vector search, and local LLM recommendations."]

    local_llm["Local LLM Server<br/><br/>OpenAI-compatible or Ollama-compatible model endpoint used to generate learner recommendations."]
    skillsfuture["Local SkillsFuture Data Files<br/><br/>Workbook and processed JSON/JSONL snapshots for roles, tasks, skills, and unique skills."]
    browser_storage["Browser Cookies<br/><br/>Stores lightweight demo auth identity for frontend routing."]

    learner -->|"Uses web dashboard"| system
    admin -->|"Runs indexing and checks APIs"| system
    system -->|"Generates grounded recommendations"| local_llm
    system -->|"Indexes and retrieves career evidence"| skillsfuture
    system -->|"Reads/writes demo session identity"| browser_storage
```

## C2: Container Diagram

```mermaid
flowchart TB
    learner["Learner / Career Switcher"]
    admin["Product Team / Demo Operator"]

    subgraph product["Job and Skills Track"]
        frontend["Next.js Frontend<br/><br/>Landing, login, register, dashboard tabs, learner intake, pathway generator, theme toggle."]
        backend["FastAPI Backend<br/><br/>Auth, users, roles, learner analysis, resume parsing, vector management, request logging, CORS."]
        sqlite["SQLite Database<br/><br/>Tortoise ORM persistence for users and role records."]
        chroma["ChromaDB Persistent Store<br/><br/>job_skills and unique_skills collections with cosine HNSW search."]
        data_files["SkillsFuture Data Files<br/><br/>Excel source files plus processed joined role JSONL and unique skills JSON."]
    end

    local_llm["Local LLM Server<br/><br/>OpenAI-compatible / Ollama-compatible completion API."]

    learner -->|"Browser requests"| frontend
    admin -->|"Indexing requests / API checks"| backend

    frontend -->|"HTTP JSON API<br/>/auth, /users, /roles/pathway, /learner/analyze"| backend
    frontend -->|"Stores demo auth email"| browser_cookies["Browser Cookies"]

    backend -->|"Reads/writes via repositories"| sqlite
    backend -->|"Similarity search and indexing"| chroma
    backend -->|"Loads source records during indexing"| data_files
    backend -->|"Prompt with retrieved evidence"| local_llm

    data_files -->|"Seed vectors"| chroma
```

## Notes

- The frontend defaults to `NEXT_PUBLIC_API_BASE_URL=http://localhost:8000`.
- The backend defaults to SQLite at `DATABASE_URL=sqlite://db.sqlite3`.
- ChromaDB is configured by `VECTOR_DB_PATH`, `VECTOR_DB_COLLECTION`, `VECTOR_DB_UNIQUE_SKILLS_COLLECTION`, and `VECTOR_DB_HNSW_SPACE`.
- Learner recommendations are grounded by vector evidence first, then generated by the configured local LLM.
